# Fluxo de reserva

## Objetivo

Coletar dados suficientes para que a operação consiga validar e seguir a reserva sem lookup manual.

## Campos da reserva

- nome completo;
- WhatsApp;
- e-mail;
- CPF opcional;
- data de nascimento opcional;
- ponto de embarque;
- observações;
- aceite dos termos.

## Pós-envio

- exibir mensagem de pré-reserva recebida;
- informar que a confirmação será manual após pagamento.

## Regras

- Reservas públicas continuam abertas para o visitante.
- O fluxo deve continuar compatível com o cookie httpOnly e o refresh já existente.
